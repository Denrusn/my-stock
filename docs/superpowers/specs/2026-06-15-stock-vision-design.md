# Stock Vision — A 股中长线趋势监控大屏

> 沉浸式大屏风格 · 深色主题 · stock-sdk 数据源 · 局域网 HTTP 服务

**版本:** v1.0
**日期:** 2026-06-15
**状态:** 设计定稿，待实施

---

## 1. 概述

### 1.1 目标

构建一个 A 股中长线趋势监控数据大屏，面向桌面浏览器和局域网投屏场景，提供大盘指数、板块轮动、北向资金、市场情绪、自选股技术面的全景可视化。

### 1.2 核心原则

- **中长线视角**：以周/月线级别趋势为主，不追求秒级实时精度
- **沉浸式大屏**：深色主题 + 发光质感 + 动态动效，适合大屏展示
- **数据驱动**：所有数据来自 stock-sdk（腾讯行情），后端聚合缓存
- **局域网服务**：单页 HTML 应用，局域网上任意设备可访问

### 1.3 使用场景

- 交易日盘后复盘
- 中长线趋势分析
- 大屏投屏展示
- 移动端浏览器查阅

## 2. 设计方向

**选定方案：C · 沉浸式大屏风格**

对比了三种方向：
| 方案 | 特点 | 选择理由 |
|------|------|---------|
| A 专业机构风格 | 高密度、类似 Wind/Bloomberg | 信息门槛较高 |
| B 现代数据叙事 | 浅色卡片、类似 Notion | 大屏不够震撼 |
| **C 沉浸式大屏** | **深色炫酷、发光质感** | **视觉效果最佳，适合大屏** |

### 3.2 区块详情

#### ① TOP BAR
- **左侧**：品牌 Logo + "STOCK VISION" 标题
- **右侧**：当前日期 + 实时状态指示灯（绿色呼吸动画 = 运行中）
- 高度：60px

#### ② 四大指数卡片
- 一行四列，等宽等高
- 每张卡片包含：指数名称、当前点位（大号字体）、涨跌幅（红绿色）、迷你走势图（近5日K线缩略）
- 涨跌幅负数为红色 (#ff5252)，正数为绿色 (#00c853)
- 指数范围：上证(sh000001)、深证(sz399001)、创业板(sz399006)、科创50(sh000688)

#### ③ 板块轮动 Top 10
- 左侧区域，占据约 55% 宽度
- 表格式排行：板块名称 | 周涨幅 | 月涨幅 | 涨跌幅柱状条
- 周涨幅为正时绿色、为负时红色
- 数据源：stock-sdk `get_industry_list` 返回的行业板块涨跌幅排序

#### ④ 北向资金面板
- 右侧上方区域
- 今日净流入（大字突出）、本周累计、本月累计
- 近 20 日资金流向热度图（横向色条，绿色=净流入，红色=净流出）
- 数据源：stock-sdk `get_northbound_flow_summary`

#### ⑤ 市场情绪面板
- 右侧下方区域
- 涨/跌家数、涨停/跌停家数、总成交额
- 可选涨跌家数环形占比图
- 数据源：stock-sdk `get_zt_pool`（涨停跌停）、指数行情衍生（涨跌家数）

#### ⑥ 自选股技术面
- 底部全宽区域
- 每只自选股一张小卡片，显示名称、当前价、周K线缩略图
- 5 只自选股：歌尔股份(002241)、福达合金(603045)、华电辽能(600396)、京东方A(000725)、湖南黄金(002155)

### 4.3 动效设计

| 动效 | 触发 | 效果 |
|------|------|------|
| 数字滚动 | 页面加载 / 切换 | CountUp 动画，从0增长到目标值 |
| 数值闪烁 | 数据更新时 | 数值变化时背景色瞬闪（绿/红），持续 1.5s |
| 卡片悬浮 | hover | 轻微上浮 (translateY(-2px)) + 边框发光 (box-shadow) |
| 状态指示灯 | 持续 | 绿色呼吸动画 (opacity 0.4→1) |
| 关键数据 | 数值为正/负 | 北向资金正数时脉动吸引注意 |

## 5. 技术架构

### 5.1 整体架构图

```
┌─────────────┐     HTTP API      ┌──────────────────┐
│  stock-sdk   │ ────────────────→ │  Node.js 后端     │
│  (数据源层)   │                   │  (聚合/缓存层)     │
│              │                   │                  │
│  · 实时行情   │                   │  · 定时轮询       │
│  · K线数据   │                   │  · 内存缓存       │
│  · 板块列表   │                   │  · JSON API输出   │
│  · 北向资金   │                   │  · 静态文件服务    │
│  · 涨停股池   │                   │                  │
└─────────────┘                   └────────┬─────────┘
                                           │ HTTP
                                           ▼
                                  ┌──────────────────┐
                                  │  HTML 大屏前端     │
                                  │  (浏览器渲染)      │
                                  │                  │
                                  │  · Chart.js 图表  │
                                  │  · 定时拉取数据    │
                                  │  · 动效渲染       │
                                  └──────────────────┘
```

### 5.2 后端设计

- **语言/运行时**：Node.js (>=18)
- **Web 框架**：原生 Node.js HTTP 模块 + 简易路由（零外部依赖）
- **缓存策略**：内存 Map，TTL 30-300 秒（不同类型不同）
- **API 端点**：

| 端点 | 缓存 TTL | 说明 |
|------|---------|------|
| `GET /api/quotes` | 30s | 四大指数 + 5只自选股实时行情 |
| `GET /api/industries` | 300s | 行业板块涨跌幅排名 Top 10 |
| `GET /api/northbound` | 300s | 北向资金汇总 |
| `GET /api/sentiment` | 30s | 市场情绪（涨跌家数等） |
| `GET /api/kline/:code` | 600s | 自选股周K线数据 |
| `GET /` | - | 静态 HTML 大屏页面 |

### 5.3 前端设计

- **渲染方案**：纯静态 HTML + CSS + JS
- **图表库**：Chart.js (CDN 引入)
- **数据拉取**：`setInterval` + `fetch`，30 秒自动轮询
- **状态管理**：全局 `dashboardState` 对象，数据更新时触发 re-render

### 5.4 数据流

```
页面加载
  ↓
fetch /api/quotes, /api/industries, /api/northbound, /api/sentiment (并行)
  ↓
CountUp 数字滚动动画展示数据
  ↓
每 30 秒自动轮询更新
  ↓
变化的数据高亮闪烁
  ↓
收盘后停止轮询，展示最终数据
```

## 6. 界面交互细节

### 6.1 响应式设计

- **主目标**：1920×1080 全屏展示
- **次目标**：1280×720 最小支持
- 使用 CSS Grid + Flexbox 弹性布局
- 缩放至 1280 时不换行，缩小内边距和字号

### 6.2 加载状态

- 首次加载时显示全屏 loading 遮罩
- 每个数据卡片内部显示独立的加载骨架
- 数据到达后依次渐入渲染

### 6.3 错误处理

- 单模块数据获取失败不影响其他模块
- 失败模块显示 "数据获取失败" + 重试按钮
- 连续 3 次失败全局弹窗提示
